Judge 0.1.1

Easy client-side form validation for Rails 3

Judge allows easy live form validation for Rails 3 apps, by porting many ActiveModel::Validation features to the client side and exposing your model validations as JSON within HTML5 data attributes.

Dependencies

The only runtime dependency Judge has is jQuery. I'll eventually offer the JavaScript element in a framework-agnostic format, but Rails 3.1 will ship with jQuery by default, so this is a good start. :)

Installation

# Gemfile
gem "judge", "~> 0.1.1"
$ cd path/to/your/app
$ bundle install

To copy the JavaScript files to your public/javascripts directory, run:

$ cd path/to/your/app
$ rails g judge

For further options (like specifying a different javascript path) run:

$ rails g judge --help

Usage

Replace form_for with validated_form_for. All arguments are the same. Replace your f.text_field or f.text_area with f.validated_text_field and f.validated_text_area respectively.

<%= validated_form_for(@post) do |f|%>
  <%= f.validated_text_field :title %>
  <%= f.validated_text_area :body %>
<% end %>

Check out example.judge.js for the simplest JavaScript binding possible:

// this binds to all validated text fields and text areas
$('[data-validate]').judge('keyup');

You can alter the selector and pass an object literal containing custom callbacks for greater control. Within the callback functions, the keyword this is a jQuery object containing the validated element. The failure callback receives an array containing validation error messages.

$('form#mygreatform input[data-validate]').judge('keyup', {
  success: function() {
    this.addClass('valid');
  },
  failure: function(errors) {
    this.addClass('invalid');
    alert(errors.join(', '));
  }
});

Tests

There are integration tests for the Judge form helpers and the JavaScript is unit tested with Jasmine.

$ cd path/to/judge

# all tests
$ rake

# only integration tests
$ rake test

# only unit tests
$ rake jasmine:ci
