Easy client-side form validation for Rails 3

Judge allows easy live form validation for Rails 3 apps, by porting many ActiveModel::Validation features to the client side and exposing your model validations as JSON within HTML5 data attributes.

Dependencies

The only dependency Judge has is jQuery. I'd like to offer Judge without any dependencies at all, but since Rails 3.1 will ship with jQuery by default, this isn't currently high on my list of priorities.

Install

$ gem install judge #(not yet published to gemcutter)

or add "judge" to your Gemfile and then

$ bundle install

To copy Judge to your public/stylesheets directory, run

$ cd path/to/your/app
$ rails g judge

For further options (like custom stylesheet paths) run

$ rails g judge --help

Usage

Replace form_for with validated_form_for. All arguments are the same. Replace your f.text_field or f.text_area with f.validated_text_field or f.validated_text_area respectively.

<%= validated_form_for(@post) do |f|%>
  <%= f.validated_text_field :title %>
  <%= f.validated_text_area :body %>
<% end %>

Check out example.judge.js for the simplest JavaScript binding possible:

// this binds to all validated text fields and text areas
$('[data-validate]').judge('keyup');

You can alter the selector and pass an object literal containing custom callbacks for greater control:

$('form#mygreatform input[data-validate]').judge('keyup', {
  success: function(element) { element.addClass('valid'); },
  failure: function(element) { element.addClass('invalid'); }
});
